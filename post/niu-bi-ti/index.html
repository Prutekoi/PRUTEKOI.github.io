JZOJ6301 普及组

去年暑假给我们考的noip模拟/cy/ruo

现在能做JZOJ了，对被当时的毒瘤计数虐记忆深刻，回去翻出来这套题。还是只想出来求逆的 O(nlogn)O(nlogn)O(nlogn)QAQ

给定一整数 xxx，有 TTT 组询问，每次给定一个 nnn，询问 n×nn\times nn×n 的矩阵中填整数，满足每行之积和每列之积都为 xxx 的方案数。其中 xxx 满足质因数分解为 ∏pc\prod p^c∏pc 后每个 ccc 都小于等于 222。T≤2×105,n≤5×106T\leq 2\times 10^5,n\leq 5\times 10^6T≤2×105,n≤5×106

首先发现填的东西关于符号和每个质因子都是独立的。
而填符号是一个经典问题，方案数为 2(n−1)22^{(n-1)^2}2(n−1)2。
然后如果某个质因子的次数为 111，也是很好算的。就是 n!n!n!。
接下来关键就是次数为 222 的了。。可以把问题简化为矩阵中填 0,1,20,1,20,1,2，使得每行每列之和都为 222。
有一个比较好想的dp。就是考虑最后一列的填数情况的话，它可能填两个 111，也可能填一个 222。然后再考虑一下就可以想到状态 f[i][j][k][l]f[i][j][k][l]f[i][j][k][l] 表示当前填 iii 列，jjj 行和为 000，kkk 行和为 111，lll 行和为 222。然后发现 i,j,k,li,j,k,li,j,k,l 是可以互相计算的，可以省掉两维。然后就是单次 O(n2)O(n^2)O(n2) 的dp。且一次dp只能算出一个 nnn。

然后我的辣鸡思路：
考虑如果只能填 0,10,10,1 的话，每一行都填了两个 111，我们把这两列对应的点连边，最后一定连出来好多环。但是要求是不能有自环。我们只要统计出这样的连边方式之后，再乘一个阶乘（每条边在哪一行连的），就是只能填 0,10,10,1 的答案了。设 fif_ifi​ 表示 iii 个点的连边方式。

fi=(i2)fi−2+∑j=3i(ij)fi−j(j−1)!2f_i={i\choose 2}f_{i-2}+\sum_{j=3}^i{i\choose j}f_{i-j}\frac{(j-1)!}{2} fi​=(2i​)fi−2​+j=3∑i​(ji​)fi−j​2(j−1)!​

长度为 222 的环没方向，而大于等于 333 的环则有方向，我们要统计的是无向的环，所以除二。
然后弄一下EGF发现这就是一个 F=F0+F∗GF=F_0+F*GF=F0​+F∗G 的形式，求逆nlogn就可以了。

题解这部分是直接设的 fif_ifi​ 表示 i×ii\times ii×i 矩阵只填 0,10,10,1 的方案，大同小异，也是推出来一个 O(nlogn)O(nlogn)O(nlogn)。

我们有了只填 0,10,10,1 的答案之后，单独考虑一下填 222 发现如果有一格填了 222，我们可以直接把这行这列删掉。然后枚举一下填了几个 222 之后也是一个卷积的式子，不再赘述。

发现数据范围这么大，可以直接猜标算做法：一遍 O(n)O(n)O(n) dp，直接求出所有 nnn 的答案。
然后我们只能设 fif_ifi​ 表示 i×ii\times ii×i 的矩阵的答案了。接下来就是神仙的构造映射。

还是考虑最后一列填的情况，如果它填了一个 222，那挺好说，可以直接删掉最后一列和所在行，fi+=i×fi−1f_i+=i\times f_{i-1}fi​+=i×fi−1​。
如果它填了两个 111，那真就不太好说了。。

这里 a,ba,ba,b 是最后一列的两个 111，aaa 与 ccc，bbb 与 ddd 同行，ccc 与 fff，ddd 与 eee 同列。
我们现在仍然是考虑删掉 a,ba,ba,b，这样才能缩小规模。这时 c,dc,dc,d 所在行都少了 111，怎么补上让它重新成为一个合法方案？直觉告诉我们应该把 c,dc,dc,d 移到同一行。发现这样确实没有影响更多的行和列，又重新成为了一个合法方案。
而 c,dc,dc,d 如果在同一列呢？那么我们也是把它们移到同一行：变成了一个 222。

发现这个转移貌似还要分情况，于是我们重新定义状态：fif_ifi​ 表示最后一列是两个 111 的方案数，gig_igi​ 表示最后一列是一个 222 的方案数。
然后 gi=i(fi−1+gi−1)g_i=i(f_{i-1}+g_{i-1})gi​=i(fi−1​+gi−1​) 不用说。
还是回到上面的讨论，融合后的这一行应该放到哪？？
发现这行放到哪其实并不重要（但是所有映射方案要放到同一行，这样才能合法地构造映射）。因为它对方案的影响相当于是钦定了某一行有两个 111 或一个 222。
然后我们就考虑写转移方程式了。先枚举一下这两个 111 在哪两行，有一个组合数。然后我们关注的就是与它们对应的 c,dc,dc,d 在哪列。如果它们在不同列，就是钦定某一行有两个 111。而它们一上一下有两种情况，所以应该乘二。而若它们在同列，则钦定了某一行有一个 222。发现钦定某行情况和钦定最后一列情况其实是等价的。
然后就有了递推式：fi=(i2)(2fi−1+gi)f_i={i \choose 2}(2f_{i-1}+g_{i})fi​=(2i​)(2fi−1​+gi​)
就可以 O(n)O(n)O(n) 递推了。

JZOJ6302 提高组

QAQ

有 TTT 组询问，每次询问给出 n,x,yn,x,yn,x,y，求 111 到 nnn 的排列，满足 px=yp_x=ypx​=y，且最长下降子序列长度最多为 222 的方案数。
T≤106,n≤107T\leq 10^6,n\leq 10^7T≤106,n≤107

首先找一下最长下降子序列长度最多为 222 的排列有什么性质。看起来没啥，那正难则反，如果一个排列具有大于等于 333 的下降子序列，那么我们随便从这个子序列的内部找一个位置，它前面有比它大的，后面有比它小的。然后我们发现这就是充要条件。

所以问题转化为我们要求对于每个数，要么它前面的都比它小，要么它后面的都比它大（比它小的都在前面）。

然后这种dp的套路就是要么从大到小枚举数要么从小到大枚举数嘛。

这里是都可以的啦，我是从大到小枚举的，式子长这样：
f[i][j]=∑k=j−1i−1f[i−1][k]f[i][j]=\sum_{k=j-1}^{i-1}f[i-1][k]f[i][j]=∑k=j−1i−1​f[i−1][k]。
其中 f[i][j]f[i][j]f[i][j] 表示现在限制只能在第 jjj 个数后及之前插入数的方案数。
因为如果插入到第 jjj 个数后，那么对后面产生的限制仍是只能在第 jjj 个数后及之前。
如果插入到最前，那么整体往后错一位。

然后考虑还有 px=yp_x=ypx​=y 的限制呢。

分类讨论一下，如果 x=yx=yx=y，那么比 xxx 小的都得在 xxx 前，大的都得在后。这时两边无关，可以分别统计，再相乘。

如果 x>yx>yx>y，发现 xxx 左边一定会有比 yyy 大的。那么我们只能限制比 yyy 小的都在 xxx 左边。那么我们到插入 yyy 之后比 yyy 小的都会插入到它左边，排名在后面增加的数目就固定了（y−1y-1y−1），那么在插入 yyy 时它的排名也应固定，使得最后它可以到达第 xxx 个。

x<yx<yx<y 的发现如果还这么dp不太容易处理了，一种方式是再写一个从小到大插数的dp。。不过不用这么傻，我们让 x=n−x+1,y=n−y+1x=n-x+1,y=n-y+1x=n−x+1,y=n−y+1，这时容易发现和原限制是等价的。又转化为 x>yx>yx>y。

我们现在有了单次 O(n2)O(n^2)O(n2) 的做法。

没啥思路，打个表吧。

发现没有 px=yp_x=ypx​=y 限制时的答案数列竟然是卡特兰数！

然后想为什么会是卡特兰数。它与网格有关系，猛然发现这dp就像是一个网格游走！

然后发现这个转移不太标准，构造一个 bi=i−aib_i=i-a_ibi​=i−ai​，它就变得很标准了。然后又发现固定 px=yp_x=ypx​=y 其实是固定了路径要经过哪个点。然后就对这个点之前和之后分别统计，再相乘即可。翻折法统计。

#include<bits/stdc++.h>
using namespace std;
const int N=2e7+50,mod=1e9+7;
int n,x,y,f[N],g[N],T,J[N],I[N],inv[N];
int C(int n,int m){return m<0||n<m?0:1ll*J[n]*I[m]%mod*I[n-m]%mod;}
int calc(int n,int m){
    if(n>=m)return (C(n+m,n)-C(n+m,n+1)+mod)%mod;
}
int dp(int n,int x,int y){
    if(!n)return 1;
    if(!x)return 1ll*C(2*n,n)*inv[n+1]%mod;
    return 1ll*(calc(n-y,n-x+1)-(n-y-1>=n-x+1?calc(n-y-1,n-x+1):0)+mod)*(C(y-1+x-1,y-1)-C(y-1+x-1,y-2)+mod)%mod;
    memset(f,0,sizeof(int)*(n+1));f[1]=1;
    for(int i=n-1;i;i--){
        int sum=0;
        if(i==y){
            x-y+1;
            for(int j=n-i;j>=x-y;j--)(sum+=f[j])%=mod;
            memset(f,0,sizeof(int)*n);f[x-y]=sum;
        }
        else{
            for(int j=n-i+1;j;j--)
                (sum+=f[j])%=mod,f[j]=(sum+f[j-1])%mod;
        }
    }
    int ret=0;
    for(int i=1;i<=n;i++)(ret+=f[i])%=mod;
    return ret;
}
int main(){
    freopen("tg.in","r",stdin);
    freopen("tg.out","w",stdout);
    scanf("%d",&T);
    J[0]=I[0]=inv[1]=1;
    for(int i=2;i<N;i++)inv[i]=mod-1ll*mod/i*inv[mod%i]%mod;
    for(int i=1;i<N;i++)J[i]=1ll*J[i-1]*i%mod,I[i]=1ll*I[i-1]*inv[i]%mod;
    while(T--){
        scanf("%d%d%d",&n,&x,&y);
        if(x==y)printf("%d\n",1ll*dp(x-1,0,0)*dp(n-x,0,0)%mod);
        else if(x>y)printf("%d\n",dp(n,x,y));
        else printf("%d\n",dp(n,n-x+1,n-y+1));
    }
    return 0;
}

###CF1172F

某场div1的最后一题，赛时无人AC。。我当时打了对应的div2/cy

有一个长度为 nnn 的序列和一个模数 ppp，定义add_mod(x,y)返回 x+y>p?x+y−p:x+yx+y>p?x+y-p:x+yx+y>p?x+y−p:x+y。有 mmm 次询问，每次给定 l,rl,rl,r，求对于初值为 z=0z=0z=0，对于在区间里的 iii 顺次执行 z=add_mod(z,a[i])，求最后 zzz 的值。
1≤n≤106,1≤m≤2×105,1≤p≤109,−109≤ai≤1091\leq n\leq 10^6,1\leq m\leq 2\times 10^5,1\leq p\leq 10^9,-10^9\leq a_i\leq 10^91≤n≤106,1≤m≤2×105,1≤p≤109,−109≤ai​≤109

考虑这种顺次考虑每个元素的形式，可以想到维护一个映射。对于一个区间有一个映射 F(x)F(x)F(x)，表示如果从左端送进一个 xxx 在右端会出来一个 F(x)F(x)F(x)。这种做法之前在一个博弈论题目中用到过，那题是维护一个状压的状态，直接用数组映射的。
但是这里显然不能直接用数组映射。那么考虑可能是一个函数的形式。然后再观察就会发现对于长度为 lenlenlen 的区间，经过这个区间最少就是不减 ppp，最多减 lenlenlen 个 ppp，发现是一个分段函数，每一段都是一次函数，且最多有 lenlenlen 段。
那么我们如果用一个线段树存每个区间的分段函数，则空间是 O(nlogn)O(nlogn)O(nlogn) 的，可以接受。
现在问题就是如何复合两个分段线性函数（线段树的信息合并）。

有一个比较显然的结论是对于 y>xy>xy>x，则 F(y)−y≥F(x)−xF(y)-y\geq F(x)-xF(y)−y≥F(x)−x，即大的数不可能减 ppp 的次数更少。单调性有了，我们只要对于每一个 kkk 求出最小的 xxx 使得 F(x)−x=k∗pF(x)-x=k*pF(x)−x=k∗p 即可，有了每一段的左端点右端点也就有了。

然后考虑这个 kkk 次是分别在左右两边减的，一个很自然的思路就是枚举左边的第 iii 段，和右边的第 jjj 段，如果能到达的话就去更新 i+ji+ji+j。但是这样合并是 n2n^2n2 的。考虑可能有很多 iii 和 jjj 不可达，那么可能可以用双指针优化。
有一个结论就是在两段分界的地方纵坐标相差 p−1p-1p−1。这个是显然的。所以看起来相交的部分并不多？

然后我们再证明每段的横坐标跨度长度至少为 ppp。我们考虑一个折线图表示到达每个位置时的 zzz 的大小。设 d=−infd=-infd=−inf，再设这个区间的最大前缀和的位置为 pospospos，则 f(d)−d=0f(d)-d=0f(d)−d=0。然后慢慢增大 ddd，直到 pospospos 所在横坐标的折线碰到了 y=py=py=p 这条直线。这时进入下一段，减了一个 ppp，pos以后的折线都向下平移 ppp 个单位。然后再增大 ddd，如果下一次仍然是 pos 碰到 y=py=py=p，则这一段的长度为 ppp，满足。否则一定在 pospospos 前面的位置碰到，然后那个位置以后的折线向下平移， pospospos 就碰不到了。直到 pospospos 再次碰到才会使得 f(d)−df(d)-df(d)−d 增大，然后段的长度就为 ppp。得证了。

所以当左边的函数两段分界的时候，可行的 jjj 最多只会减 111。然后就可以双指针了。

#include<bits/stdc++.h>
#define LL long long
#define pb push_back
using namespace std;
const int N=1e6+50;
const LL inf=1e16;
int n,m,p;LL now,sum[N];
vector<LL>v[N*4];
int read(){
	int x=0,c,f=1;
	while(!isdigit(c=getchar()))c=='-'?f=-1:0;
	while(isdigit(c))x=x*10+c-48,c=getchar();
	return x*f;
}
void up(int x,LL S){
	int L=x<<1,R=x<<1|1;
	v[x].resize(v[L].size()+v[R].size()-1,inf);
	for(int i=0,j=0;i<v[L].size();i++){
		LL xl=v[L][i],xr=i==v[L].size()-1?inf:v[L][i+1]-1,yl=xl+S-1ll*i*p,yr=xr+S-1ll*i*p;
		if(j)j--;
		while(j<v[R].size()-1&&v[R][j+1]<=yl)j++;
		for(;j<v[R].size()&&v[R][j]<=yr;j++)v[x][i+j]=min(v[x][i+j],max(xl,v[R][j]-S+1ll*i*p));
	}
	v[x][0]=-inf;
}
void build(int x,int l,int r){
	if(l==r){sum[l]=sum[l-1]+read(),v[x].pb(-inf);v[x].pb(p-(sum[l]-sum[l-1]));return;}
	int mid=(l+r)>>1;
	build(x<<1,l,mid);build(x<<1|1,mid+1,r);
	up(x,sum[mid]-sum[l-1]);
}
void query(int x,int l,int r,int ql,int qr){
	if(l>=ql&&r<=qr){now+=sum[r]-sum[l-1]-1ll*(upper_bound(v[x].begin(),v[x].end(),now)-v[x].begin()-1)*p;return;}
	int mid=(l+r)>>1;
	if(ql<=mid)query(x<<1,l,mid,ql,qr);
	if(qr>mid)query(x<<1|1,mid+1,r,ql,qr);
}
int main(){
	n=read();m=read();p=read();
	build(1,1,n);
	for(int i=1,l,r;i<=m;i++){
		l=read();r=read();
		now=0;query(1,1,n,l,r);
		printf("%lld\n",now);
	}
	return 0;
}
